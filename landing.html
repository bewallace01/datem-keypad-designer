The Problem

Macro syntax is brutal.
You shouldn't have to memorize it.

Setting up a DAT/EM keypad means writing macros by hand in a keystroke-injection format. Every token matters: {RET} sends Enter, _ makes commands language-independent, - suppresses dialogs (critical — a popped dialog interrupts your active drawing command), and ' makes a command transparent. (No ^C^C; the keypad doesn't honor it.)

For the operator who lives in this syntax, fine. For the rest of us, it's painful. Mistakes are silent and only surface when the button doesn't work the way you want during a stereo session.

So: describe the button in plain English. Get correct syntax back. Move on.

You write
"switch to ROAD_EOP layer
and start a 3D polyline"
↓
It generates
-LAYER{RET}SET{RET}ROAD_EOP{RET}{RET}_3DPOLY
Features

Built for stereo compilation workflow.

The AI knows DAT/EM Summit keywords, AutoCAD Civil 3D commands, OSNAP transparency, and the rules of stereo digitizing — so the macros it produces actually work mid-session.

✨

Plain English → macros

Describe a button. Get correctly-formatted DAT/EM macros back, with a label, color category, and an explanatory note.

📋

Per-project context

Paste your feature code library once. The AI uses your real layer names — ROAD_EOP, BREAKLINE, HYDRO_EDGE — instead of generic placeholders.

⚡

Auto-fill layouts

Generate a complete keypad in one click. Summit controls up top, drawing tools, layer switches by feature class, OSNAPs at the bottom. Smart pairings (BREAKLINE auto-starts a feature line).

🗂

Project switching

One layout per job. Switch between them without losing work. Duplicate a project to seed a new one with similar conventions.

🎨

Color-coded categories

Summit, AutoCAD, Capture, OSNAP, layer switch, mixed macro. Visual grouping so the operator's eye lands on the right button fast.

📄

Reference export

Print-ready text reference grouped by category, plus JSON backups. Side-by-side with the actual DAT/EM Keypad Editor for fast data entry.

How it works

Three steps from blank to ready.

Set up project context

Paste your feature code library — BLDG, ROAD_EOP, CURB, etc. — with what each represents. The AI uses these exact names everywhere.

Auto-fill or hand-craft

Click ✨ Auto-fill to generate a complete starter layout. Or click any button to add it manually with AI-assisted command generation.

Export and apply

Export a clean reference doc grouped by category. Open it on a second monitor next to the DAT/EM Keypad Editor and copy buttons across.
